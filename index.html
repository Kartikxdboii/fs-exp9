Experiment 9 — Docker & Docker Compose

Containerize and run a complete full-stack application

🐳 Part (a): Dockerfile for Spring Boot Backend

Multi-stage build — Maven builds the JAR, then we run it in a slim JRE image

💡 Multi-stage build keeps the final image small (~300MB vs ~800MB) because Maven and source code are discarded.
📄 backend/Dockerfile
# Stage 1: Build the JAR using Maven
FROM maven:3.9-eclipse-temurin-17 AS build

WORKDIR /app

# Copy pom.xml first (caches dependencies)
COPY pom.xml .
RUN mvn dependency:go-offline

# Copy source code and build
COPY src ./src
RUN mvn clean package -DskipTests

# Stage 2: Run the JAR
FROM eclipse-temurin:17-jre

WORKDIR /app

COPY --from=build /app/target/*.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "app.jar"]
☕ HelloController.java (sample)
package com.example.demo;

import org.springframework.web.bind.annotation.*;

@RestController
public class HelloController {

    @GetMapping("/api/hello")
    public String hello() {
        return "Hello from Spring Boot in Docker!";
    }
}

⚡ Commands to Build & Run

  1. Build the image
    $ docker build -t my-backend .
  2. Run the container
    $ docker run -p 8080:8080 my-backend
  3. Test it
    $ curl http://localhost:8080/api/hello
✅ You should see: "Hello from Spring Boot in Docker!"

🌐 Part (b): Dockerfile for React Frontend

Build React with Node, then serve the production files via Nginx

💡 Node builds the app, then only the static files (~5MB) go into the Nginx image. Fast and tiny!
📄 frontend/Dockerfile
# Stage 1: Build React app
FROM node:18-alpine AS build

WORKDIR /app

COPY package.json package-lock.json ./
RUN npm install

COPY . .
RUN npm run build

# Stage 2: Serve with Nginx
FROM nginx:alpine

COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
⚙️ frontend/nginx.conf
server {
    listen 80;
    server_name localhost;

    location / {
        root   /usr/share/nginx/html;
        index  index.html;
        try_files $uri $uri/ /index.html;
    }

    # Proxy API calls to backend container
    location /api/ {
        proxy_pass http://backend:8080;
    }
}

🚀 Run Frontend & Backend Separately

  1. Build both images 
    $ docker build -t my-backend ./backend
$ docker build -t my-frontend ./frontend
  2. Create a shared network
    $ docker network create myapp-net
  3. Run backend
    $ docker run -d --name backend --network myapp-net -p 8080:8080 my-backend
  4. Run frontend
    $ docker run -d --name frontend --network myapp-net -p 3000:80 my-frontend
  5. Open browser → http://localhost:3000
⚠️ Use --network so frontend (Nginx) can reach backend:8080 by container name.

🐙 Part (c): Docker Compose — Full Stack

Run frontend + backend + database together with one command

💡 Docker Compose handles networking, startup order, and environment variables automatically.

→
☕
Backend

🗄️
Database
:3306 → MySQL
📄 docker-compose.yml
version: "3.8"

services:

  # MySQL Database
  db:
    image: mysql:8
    container_name: myapp-db
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: root123
      MYSQL_DATABASE: myappdb
      MYSQL_USER: appuser
      MYSQL_PASSWORD: apppass123
    ports:
      - "3306:3306"
    volumes:
      - db-data:/var/lib/mysql
    networks:
      - myapp-network

  # Spring Boot Backend
  backend:
    build: ./backend
    container_name: myapp-backend
    restart: always
    ports:
      - "8080:8080"
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://db:3306/myappdb
      SPRING_DATASOURCE_USERNAME: appuser
      SPRING_DATASOURCE_PASSWORD: apppass123
    depends_on:
      - db
    networks:
      - myapp-network

  # React Frontend (Nginx)
  frontend:
    build: ./frontend
    container_name: myapp-frontend
    restart: always
    ports:
      - "3000:80"
    depends_on:
      - backend
    networks:
      - myapp-network

networks:
  myapp-network:
    driver: bridge

volumes:
  db-data:
⚙️ application.properties (Spring Boot)
# Values come from Docker Compose environment variables
spring.datasource.url=${SPRING_DATASOURCE_URL}
spring.datasource.username=${SPRING_DATASOURCE_USERNAME}
spring.datasource.password=${SPRING_DATASOURCE_PASSWORD}
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true

⚡ Docker Compose Commands

  1. Start everything
    $ docker-compose up --build
  2. Check status
    $ docker-compose ps
  3. View logs
    $ docker-compose logs backend
  4. Stop everything
    $ docker-compose down

✅ Verify in Browser

  1. Frontend: Open http://localhost:3000 — see the React app
  2. Backend API: Open http://localhost:8080/api/hello — see Spring Boot response
  3. Full flow: React → Nginx proxies to backend → backend queries MySQL → response shown
✅ If all 3 work, your full-stack Docker setup is complete!
